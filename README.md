# 설비/품질 관리 시스템

기존에 엑셀로 관리하던 두 가지 업무를 하나의 웹앱으로 통합했습니다.

- **생산일지 입력**: 설비가동정보(교대별 설비 셋팅, 가동시간, LNG 사용량 등)
- **QC측정 입력**: 비료시료 강도테스트(시료 20개 경도값, 생산조건, 수분)
- **통합 대시보드**: 두 데이터를 날짜·조(주/야) 기준으로 자동 연동해 집계, 추이 차트, 이상 알림,
  월간 리포트를 제공

## 왜 이렇게 만들었는가

두 엑셀 파일은 별개가 아니라 한 방향으로 연결되어 있었습니다. QC측정에서 하루 여러 번 측정한
경도/수분 값을, 생산일지에는 교대별 평균으로 손으로 옮겨 적고 있었습니다. 그래서:

- 입력 화면은 실제 업무 흐름대로 **분리**했습니다 (생산일지 담당자 / QC 담당자가 서로 다른
  시점에 입력).
- 데이터는 **하나로 연동**해서, 생산일지를 입력할 때 해당 날짜·조의 QC 평균값이 자동으로
  참고/적용되도록 했습니다 (이중 입력 제거).
- 여기에 이상 알림(기준치 이탈), 가동 효율(가스사용량/시간), 월간 자동 리포트 기능을 추가했습니다.

## 시작하기

```bash
npm install
npm run dev
```

브라우저에서 http://localhost:3000 접속. 최초 접속 시 `data/app.db` (SQLite)가 자동 생성됩니다.

운영 서버(사내 PC 등)에 올릴 때는:

```bash
npm run build
npm start
```

## 화면 구성

| 경로          | 설명                                                         |
| ------------- | ------------------------------------------------------------ |
| `/dashboard`  | 통합 대시보드 (기본 화면)                                     |
| `/production` | 생산일지 입력 (교대별 설비가동정보)                            |
| `/qc`         | QC측정 입력 (비료시료 강도테스트)                              |
| `/import`     | 기존에 쓰던 엑셀 파일을 그대로 업로드해 과거 데이터를 가져오기 |

## 데이터베이스

`better-sqlite3` 기반 파일 DB(`data/app.db`)를 사용합니다. 별도 DB 서버 설치가 필요 없고,
파일만 백업하면 데이터가 보존됩니다. (`data/*.db`는 git에 커밋되지 않습니다.)

## 이상 알림 / 기준값

대시보드 우측 하단에서 경도·수분·가동시간당 가스사용량의 상/하한 기준값을 직접 조정할 수 있고,
기준을 벗어나면 자동으로 알림이 표시됩니다.
